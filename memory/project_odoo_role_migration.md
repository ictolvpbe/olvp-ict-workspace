Uiteindelijk: (1) deps in de env-pattern-image gebakken via `podman exec -u root … pip3 install --break-system-packages google-api-python-client google-auth weasyprint zeep` (apt Pango/Cairo-libs zaten er al) + `podman commit <zelfde image-tag>`; (2) `-u all` als **one-off `podman run`** met de serving-Odoo **gestopt** (anders `SerializationFailure` door concurrente registry-load). ⚠️ **vzdump-restore-gotcha**: een VM-backup terugzetten reverteert ook de **step-ca-certs (24u)** naar de backup-datum → verlopen → HAProxy-backend `verify required` weigert ze → **503** ondanks gezonde Odoo. ⚠️ **cert-bestandsnaam**: de Caddy-certs heten **`<fqdn>.olvp.be.crt`** (niet `<kort>.crt`); verse uitgifte moet exact dat pad zijn. ⚠️ **per-bestand bind-mounts**: na `step ca certificate --force` moet de **caddy-container gerecreëerd** worden (`podman rm -f caddy && systemctl restart caddy.service`), anders blijft de oude inode gemount.
